# Code of Conduct

Be respectful. Keep participation safe and professional.

## Expected

- Be kind and direct.
- Respect different experience levels and viewpoints.
- Accept constructive feedback.
- Take responsibility for mistakes.
- Focus on project quality and user safety.

## Not Accepted

- Harassment.
- Sexualized language or imagery.
- Insults, slurs, personal attacks, or political attacks.
- Publishing private information without consent.
- Repeated disruption.
- Any conduct inappropriate for a professional project space.

## Scope

Applies in:

- Issues.
- Pull requests.
- Discussions.
- Project chat.
- Official project representation.

## Enforcement

Report incidents privately to repository admins.

Preferred channel: GitHub Security Advisory.

Admins may:

- Warn.
- Remove content.
- Limit interaction.
- Temporarily ban.
- Permanently ban.

Reports are handled with privacy and good faith.
